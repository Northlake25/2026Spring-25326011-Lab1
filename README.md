# 猜诗词游戏

一个基于Wordle机制的中文诗词猜谜游戏，使用C++和Qt6开发。

## 游戏规则

1. 系统从题库中随机挑选一句诗，并提示上下句字数
2. 玩家输入诗句并点击"猜测"（无需输入标点）
3. 若诗句与答案完全匹配，则游戏成功
4. 否则：
   - 若诗句在诗词库中且字数正确，则用颜色标记反馈：
     - **绿色**：答案中含有且位置正确的字
     - **黄色**：答案中含有但位置不正确的字
     - **灰色**：答案中不存在的字
   - 若不在诗词库中或字数不正确，提示"请输入一句字数正确的古诗词"
5. 玩家每轮有20次猜测机会

## 功能特性

- ✅ Wordle式颜色反馈机制
- ✅ 题库和诗词库管理
- ✅ 灰色字提示功能
- ✅ 句间标点规则处理
- ✅ 特殊情况标记支持

## 构建要求

- C++17兼容编译器（GCC 7+、Clang 5+、MSVC 2017+）
- CMake 3.16+
- Qt 6.x

## 构建步骤

```bash
# 创建构建目录
mkdir build && cd build

# CMake配置
cmake ..

# 编译
cmake --build .

# 运行
./guess_poetry_game  # Linux/macOS
guess_poetry_game.exe  # Windows
```

## 使用说明

### 游戏界面

1. 启动游戏后自动开始新游戏
2. 在输入框中输入猜测的诗句
3. 点击"猜测"按钮或按回车提交
4. 观察颜色反馈，继续猜测直到成功或次数用尽

### 管理界面

1. 点击菜单栏"视图" -> "管理界面"
2. 在"题库管理"标签页添加/删除题目
3. 在"诗词库管理"标签页添加/删除诗词

### 添加题目

- **ID**：唯一标识符（如q001）
- **内容**：诗句内容（不含标点）
- **位置**：上句或下句
- **难度**：1-5级

### 添加诗词

- **ID**：唯一标识符（如p001）
- **标题**：诗词标题
- **作者**：作者姓名
- **朝代**：朝代信息
- **诗句**：用逗号分隔的诗句列表

## 技术架构

- **数据模型层**：Question、Poem、GameState、Feedback
- **数据访问层**：FileStorage、QuestionBank、PoemBank
- **业务逻辑层**：Validator、FeedbackProcessor、GameController
- **用户界面层**：MainWindow、GameWidget、AdminWidget

## 核心算法

### Wordle反馈算法

1. 标准化输入文本（去除标点）
2. 验证字数和诗词库存在性
3. 第一遍扫描：标记绿色（位置正确）
4. 第二遍扫描：标记黄色和灰色
