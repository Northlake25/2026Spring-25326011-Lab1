# 猜诗词游戏

一个基于Wordle机制的中文诗词猜谜游戏，使用C++和Qt6开发。

提示：本项目使用C++语言，并非基于Harmony平台开发，主要原因是本项目后期工程量较大，模拟器的运行内存难以满足调试需求。因考虑到本项目仍然贴合本课程的核心“开源软件开发”，所以也同第一次作业一并提交到了本课程组中（第一次作业在master分支下）。

BTW：本项目提交至课程组的另一个目的是，以后上课运行本游戏，就是调试程序，而不能被称作开小差了。😄

## 游戏规则

1. 系统从题库中随机挑选一句诗，并提示上下句字数
2. 玩家输入诗句并点击"猜测"（无需输入标点）
3. 若诗句与答案完全匹配，则游戏成功
4. 否则：
   - 若诗句在诗词库中且字数正确，则用颜色标记反馈：
     - **绿色**：答案中含有且位置正确的字
     - **黄色**：答案中含有但位置不正确的字
     - **灰色**：答案中不存在的字
   - 若不在诗词库中或字数不正确，提示"请输入一句字数正确的古诗词"
5. 玩家每轮有20次猜测机会

## 核心文件位置

-运行程序：bin目录下guess_poetry_game.exe
-诗词库：在bin/resources目录下的poems.json
-题库：在bin/resources目录下的questions.json

## 使用说明

### 游戏界面

1. 启动游戏后自动开始新游戏
2. 在输入框中输入猜测的诗句
3. 点击"猜测"按钮或按回车提交
4. 观察颜色反馈，继续猜测直到成功或次数用尽

### 管理界面

点击菜单栏"视图" -> "管理界面"

#### 题库管理

##### 添加题目
 
 1. 切换到"题库管理"标签页
 2. 在"诗句"输入框输入：
    - 单句：`床前明月光，疑是地上霜`
    - 批量：`床前明月光，疑是地上霜。举头望明月，低头思故乡`
 3. 设置难度（可选）
 4. 点击"添加题目"
 
 ##### 分类管理
 
 1. **创建分类**
    - 点击"创建分类"
    - 输入分类名（如：唐诗、宋词、五言、七言）
 
 2. **分配题目到分类**
    - 在题目列表中多选题目（Ctrl+点击）
    - 点击"分配到分类"
    - 选择目标分类

 3. **从分类移除**
    - 选择题目
    - 点击"从分类移除"
 
 ##### 编辑题目
 
 1. 选择要修改的题目
 2. 点击"修改题目"
 3. 修改内容、难度或分类
 4. 点击确定保存

#### 诗词库管理

##### 添加诗词
 
 1. 切换到"诗词库管理"标签页
 2. 填写诗词信息：
    - 标题：`静夜思`
    - 作者：`李白`
    - 朝代：`唐`
    - 诗句：每行一句或用逗号分隔
 3. 点击"添加诗词"
 
 ##### 编辑诗词
 
 1. 选择要修改的诗词
 2. 点击"修改诗词"
 3. 修改信息
 4. 点击确定保存

#### 导出数据
 
 1. 点击"导出数据"按钮
 2. 选择保存位置（默认：文档目录）
 3. 数据保存为：
    - `guess_poetry_data.questions.json`
    - `guess_poetry_data.poems.json`
 
#### 导入数据
 
 1. 点击"导入数据"按钮
 2. 选择导入文件
 3. 确认替换当前数据

## 技术架构

- **数据模型层**：Question、Poem、GameState、Feedback
- **数据访问层**：FileStorage、QuestionBank、PoemBank
- **业务逻辑层**：Validator、FeedbackProcessor、GameController
- **用户界面层**：MainWindow、GameWidget、AdminWidget

## 核心算法

### Wordle反馈算法

1. 标准化输入文本（去除标点）
2. 验证字数和诗词库存在性
3. 第一遍扫描：标记绿色（位置正确）
4. 第二遍扫描：标记黄色和灰色
